Understanding Radix Sort

Algorithm Theory and Step-by-Step Analysis. An educational walkthrough of non-comparison-based sorting.

1 Introduction

Radix Sort is a unique sorting technique that differs significantly from standard algorithms like Bubble Sort, Merge Sort, or Quick Sort.

  • Non-Comparison Based: Unlike other algorithms that compare two elements (e.g., is $A > B$?), Radix Sort does not compare elements directly to determine their order.
  • Digit-by-Digit Sorting: It sorts numbers by processing individual digits.
  • Direction: It typically works from the Least Significant Bit (LSB) to the Most Significant Bit (MSB) (i.e., from the unit place to the hundreds/thousands place).

2 Working Principle

The algorithm follows a systematic approach to sort a list of integers:

  1. Identify Maximum Digits: Find the number with the maximum number of digits ($d$).
  2. Pad with Zeros: Add leading zeros to all numbers so they all have exactly $d$ digits.
  3. Sort by Position: Start from the rightmost digit (Unit place) and move left:
    • Pass 1: Sort based on Unit place.
    • Pass 2: Sort based on Tens place.
    • Pass $d$: Sort based on the Most Significant place.

Crucial Concept: Stability

During each pass, if two numbers have the same digit at the current position, their relative order must remain unchanged. This property is known as Stability.

4 Analysis of Radix Sort

Time Complexity

The time complexity of Radix Sort depends on the number of digits ($d$) and the number of elements ($n$). Since it typically uses a stable intermediate sort (like Counting Sort) for each digit:

$\text{Time Complexity} = O(d \times (n+b))$
  • $d$: The number of digits in the maximum element.
  • $n$: The number of elements in the array.
  • $b$: The base of the number system (e.g., 10 for decimal numbers).

Note: If $d$ is constant and $b$ is small enough (e.g., $b=10$), Radix Sort operates in Linear Time $O(n)$, making it asymptotically faster than comparison-based sorts for specific datasets.

Space Complexity

Radix Sort requires auxiliary space for the buckets (based on the base $b$) and the output array during the intermediate sorting steps.

$\text{Space Complexity} = O(n+b)$
