1. Concept Overview

The Floyd-Warshall algorithm is a classic Dynamic Programming approach used to solve the All-Pairs Shortest Path problem. It computes the shortest distance between every pair of vertices in a weighted graph (which can be directed or undirected, with positive or negative edge weights, provided there are no negative weight cycles).

Key Definition: The Recurrence Relation

The algorithm iteratively updates a distance matrix D. In each iteration k, it checks if routing a path through an intermediate vertex k yields a shorter distance than the currently known shortest path between vertices i and j:

D(k)[i][j] = min( D(k-1)[i][j] , D(k-1)[i][k] + D(k-1)[k][j] )

Meaning: The shortest path from i to j using only vertices from the subset {1, 2, ..., k} as intermediate points.

2. Complexity Analysis

Time Complexity: O(|V|³)

The algorithm relies on three nested loops to systematically evaluate every possible path combination across the graph:

  • Outer loop (k): Iterates through intermediate vertices (|V| times).
  • Middle loop (i): Iterates through source vertices (|V| times).
  • Inner loop (j): Iterates through destination vertices (|V| times).

Space Complexity: O(|V|²)

The primary memory requirement is the 2D distance matrix used to store shortest path values. A standard implementation updates distances in-place within a single |V| × |V| matrix to maximize memory efficiency.
