Bucket Sort Algorithm

Interactive Teaching & Complexity Analysis Tool

Target Range: [0.0, 1.0)

1 Introduction

Bucket Sort is a non-comparison based sorting algorithm. Unlike Quick Sort or Merge Sort, it sorts without direct element-to-element comparison for initial placement. It operates alongside Radix Sort and Counting Sort.

2 Prerequisites & Constraints

Data Type & Range: Operates on floating-point numbers in the range [0.0, 1.0).

Data Distribution: Input elements must be uniformly and independently distributed for optimal efficiency.

3 The Algorithm (Pseudocode)

// BucketSort(A)
1. n = length[A]
2. For i = 0 to n-1:
3.     Make B[i] an empty list
4. For i = 1 to n:
5.     index = floor(n * A[i])
6.     Insert A[i] into list B[index]
7. For i = 0 to n-1:
8.     Sort list B[i] (Insertion Sort)
9. Concatenate B[0]...B[n-1]
10. Return concatenated list

4 Complexity Analysis

Best & Average Case: O(n)

Occurs with uniform distribution. Initializing buckets O(n) + mapping O(n) + sorting tiny buckets O(n) + concatenation O(n) = O(n).

Worst Case: O(n2)

Occurs when inputs are highly clustered (e.g., all mapped to one bucket). Degenerates into Insertion Sort, yielding O(n2).
